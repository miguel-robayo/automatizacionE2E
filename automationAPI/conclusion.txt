CONCLUSIONS

  The functional automation flow for the Pet API was successfully implemented, validating:

    - Data creation
    - Data persistence
    - Data update
    - Retrieval by filters

  During execution, the following findings were identified:

    - Incorrect use of the endpoint "/pet/findPetsByStatus" - "/pet/findByStatus"
    - HTTP error handing issues - 404 errors and NumberFormatException when the backend attempts to interpret the path as
    an ID, generating unclear functional error response

  Technical level findings:

    - Conflict between Java versions (1.8 and 11/17)
    - Maven using an incorrect JAVA_HOME configuration

  Benefits of using the Karate Framework

    - Ease of creating and executing test scenarios
    - no need for additional libraries such as RestAssured
    - Simple JSON validation using match
    - Use of Background for baseUrl configuration
    - Clear separation between runner and feature files

  Improvement opportunities

    - Implementation JSON schema validation
    - Generate dynamic test data
    - Integrate with CI/CD pipelines
